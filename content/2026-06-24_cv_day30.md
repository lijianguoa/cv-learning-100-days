## 二、历史背景

### 2.1 目标检测的发展脉络

目标检测的历史可以追溯到20世纪60年代，经历了几个重要的发展阶段：

- **1960s-1990s: 萌芽期**
  - 1966年，MIT的[Marvin Minsky](https://en.wikipedia.org/wiki/Marvin_Minsky)在暑期项目中要求学生构建一个能够识别场景中物体的系统，这被视为计算机视觉目标检测的最早起源。
  - 1980年代，[David Marr](https://en.wikipedia.org/wiki/David_Marr_(neuroscientist))提出了视觉计算理论，为目标检测奠定了理论基础。

- **1990s-2000s: 特征工程时代**
  - 1990年代末，[Viola-Jones检测器](https://en.wikipedia.org/wiki/Viola%E2%80%93Jones_object_detection_framework)（2001年）的出现标志着实时目标检测的突破，首次实现了实时人脸检测。
  - 2005年，[Dalal和Triggs](https://en.wikipedia.org/wiki/Histogram_of_oriented_gradients)提出HOG特征，结合SVM成为行人检测的标准方法。

- **2012-2014: 深度学习革命**
  - [AlexNet](https://en.wikipedia.org/wiki/AlexNet)在2012年ImageNet竞赛中的胜利，开启了深度学习在计算机视觉中的应用浪潮。
  - 2014年，[R-CNN](https://arxiv.org/abs/1311.2524)的提出首次将CNN引入目标检测，开启了基于深度学习的目标检测时代。

- **2015至今: 实时与精度并进**
  - [YOLO](https://arxiv.org/abs/1506.02640)（2015）和[SSD](https://arxiv.org/abs/1512.02325)（2016）实现了端到端的实时检测。
  - [Faster R-CNN](https://arxiv.org/abs/1506.01497)（2015）引入了区域提议网络(RPN)，大幅提升了检测速度和精度。

## 三、核心概念

### 3.1 什么是目标检测？

**目标检测(Object Detection)** 是计算机视觉中的一项核心任务，旨在识别图像中所有感兴趣的目标，并确定它们的位置（通常用边界框表示）。与图像分类（只需判断图像类别）不同，目标检测需要同时回答两个问题：

1. **是什么？** —— 目标的类别（如：猫、狗、汽车）
2. **在哪里？** —— 目标的位置（边界框坐标：x, y, width, height）

> **关键区别**：
> - 图像分类：输入一张图片 → 输出一个类别标签
> - 目标检测：输入一张图片 → 输出多个 `[类别, 边界框]` 对

### 3.2 目标检测的主要挑战

目标检测面临诸多挑战，理解这些挑战有助于我们设计更好的检测算法：

| 挑战 | 描述 | 示例 |
|------|------|------|
| **尺度变化** | 目标在图像中可能呈现不同大小 | 远处的小汽车 vs 近处的大卡车 |
| **遮挡** | 目标被其他物体部分遮挡 | 人群中被遮挡的行人 |
| **形变** | 非刚性物体的姿态变化 | 伸展 vs 蜷缩的猫 |
| **光照变化** | 不同光照条件下的外观差异 | 白天 vs 夜晚的场景 |
| **背景干扰** | 目标与背景相似度高 | 雪地里的白熊 |
| **多类别** | 需要同时检测多种目标 | 道路场景中的车辆、行人、交通标志 |
| **实时性** | 需要在有限时间内完成检测 | 自动驾驶需要30+ FPS |

### 3.3 滑动窗口检测 (Sliding Window Detection)

**滑动窗口检测**是最直观的目标检测方法，其核心思想是：在图像上以不同尺度和位置滑动一个固定大小的窗口，对每个窗口区域进行分类。

#### 算法流程

```
1. 定义一系列不同大小的窗口（如：64×64, 128×128, 256×256）
2. 对每个窗口大小：
   a. 在图像上以固定步长（stride）滑动窗口
   b. 提取窗口内的特征（如HOG、SIFT）
   c. 使用分类器（如SVM）判断窗口是否包含目标
3. 对所有检测到的窗口进行非极大值抑制(NMS)去除重叠框
```

#### 优缺点分析

| 优点 | 缺点 |
|------|------|
| 原理简单直观 | 计算量巨大，窗口数量随图像尺寸指数增长 |
| 实现容易 | 需要多尺度检测，进一步增加计算量 |
| 理论基础清晰 | 固定比例的窗口难以适应形变目标 |
| | 大量背景窗口导致类别不平衡 |

**计算复杂度分析**：
对于一个 $W \times H$ 的图像，窗口大小为 $w \times h$，步长为 $s$，则窗口数量为：
$$N = \frac{W - w}{s} \times \frac{H - h}{s} \times S$$
其中 $S$ 是尺度数量。对于典型设置，$N$ 可达数十万甚至上百万。

### 3.4 选择性搜索 (Selective Search)

**选择性搜索(Selective Search)** 由[Uijlings等人](https://ivi.fnwi.uva.nl/isis/publications/2013/UijlingsIJCV2013/UijlingsIJCV2013.pdf)于2013年提出，是一种基于图像分割的区域提议算法。其核心思想是：利用图像的颜色、纹理、大小等特征，通过层次化分组生成可能包含目标的候选区域，从而大幅减少需要分类的窗口数量。

#### 算法原理

选择性搜索采用**自底向上**的分组策略：

```
1. 使用Felzenszwalb算法对图像进行初始过分割，得到小区域
2. 计算相邻区域之间的相似度（颜色、纹理、大小、填充）
3. 合并相似度最高的相邻区域
4. 重复步骤2-3，直到整个图像成为一个区域
5. 在每次合并过程中，将当前所有区域作为候选提议区域
```

#### 相似度度量

选择性搜索综合考虑四种相似度：

| 相似度类型 | 描述 | 公式 |
|-----------|------|------|
| **颜色相似度** $s_{color}$ | 基于L1距离的颜色直方图相似度 | $s_{color}(r_i, r_j) = \sum_{k=1}^{n} \min(c_i^k, c_j^k)$ |
| **纹理相似度** $s_{texture}$ | 基于SIFT-like特征的纹理相似度 | $s_{texture}(r_i, r_j) = \sum_{k=1}^{n} \min(t_i^k, t_j^k)$ |
| **大小相似度** $s_{size}$ | 优先合并小区域，防止大区域吞并小区域 | $s_{size}(r_i, r_j) = 1 - \frac{size(r_i) + size(r_j)}{size(im)}$ |
| **填充相似度** $s_{fill}$ | 衡量区域合并后的紧密程度 | $s_{fill}(r_i, r_j) = 1 - \frac{size(BB_{ij}) - size(r_i) - size(r_j)}{size(im)}$ |

最终相似度为：$s(r_i, r_j) = a_1 s_{color} + a_2 s_{texture} + a_3 s_{size} + a_4 s_{fill}$

#### 选择性搜索的优势

- 将候选区域从数十万个减少到约2000个
- 召回率(Recall)高达99%（IoU>0.5时）
- 相比滑动窗口，大幅减少了计算量
- 为R-CNN系列方法奠定了基础

### 3.5 区域提议网络 (Region Proposal Network, RPN)

**区域提议网络(RPN)** 是[Faster R-CNN](https://arxiv.org/abs/1506.01497)的核心创新，首次将区域提议与目标检测整合到一个统一的深度网络中，实现了端到端的训练。

#### RPN的核心思想

RPN将区域提议问题转化为一个**回归问题**：

1. **锚框(Anchor)**：在特征图的每个位置预设一组不同尺度和长宽比的参考框
2. **二分类**：判断每个锚框是前景(包含目标)还是背景
3. **边界框回归**：对前景锚框进行位置微调

#### 锚框(Anchor)机制

锚框是RPN的核心设计：

| 参数 | 典型设置 | 说明 |
|------|---------|------|
| 尺度(Scales) | 128, 256, 512 | 相对于原图的锚框面积 |
| 长宽比(Ratios) | 1:1, 1:2, 2:1 | 锚框的形状比例 |
| 每位置锚框数 | 9 (3×3) | 每个特征图位置预设9个锚框 |

对于特征图大小为 $W' \times H'$，总锚框数量为 $W' \times H' \times 9$。

#### RPN网络结构

```
输入图像 → CNN特征提取 → 特征图
                           ↓
                    ┌──────┴──────┐
                    ↓             ↓
              1×1卷积(cls)    1×1卷积(reg)
              (2×9=18通道)    (4×9=36通道)
                    ↓             ↓
              前景/背景分类    边界框偏移量
                    ↓             ↓
                    └──────┬──────┘
                           ↓
                    非极大值抑制(NMS)
                           ↓
                    输出Top-N区域提议
```

#### RPN的损失函数

$$L(\{p_i\}, \{t_i\}) = \frac{1}{N_{cls}} \sum_i L_{cls}(p_i, p_i^*) + \lambda \frac{1}{N_{reg}} \sum_i p_i^* L_{reg}(t_i, t_i^*)$$

其中：
- $p_i$：锚框$i$为前景的预测概率
- $p_i^*$：真实标签（1前景，0背景）
- $t_i$：预测的边界框偏移量
- $t_i^*$：真实的边界框偏移量
- $L_{cls}$：交叉熵损失
- $L_{reg}$：Smooth L1损失

#### RPN vs 选择性搜索

| 特性 | 选择性搜索 | RPN |
|------|-----------|-----|
| **速度** | CPU上约2秒/图 | GPU上约10毫秒/图 |
| **召回率** | ~99% (IoU>0.5) | ~95% (IoU>0.5) |
| **是否可学习** | 否，固定算法 | 是，端到端训练 |
| **与检测网络关系** | 分离的预处理步骤 | 共享特征，统一网络 |
| **代表方法** | R-CNN, SPP-Net | Faster R-CNN |

### 3.6 非极大值抑制 (Non-Maximum Suppression, NMS)

NMS是目标检测中去除冗余检测框的标准后处理步骤：

```python
# NMS伪代码
1. 按置信度对所有检测框排序
2. 选择置信度最高的框，加入结果列表
3. 计算该框与其余所有框的IoU
4. 移除IoU > 阈值的框（视为同一目标）
5. 重复步骤2-4，直到没有剩余框
```

**IoU (Intersection over Union)** 计算公式：
$$IoU = \frac{Area_{intersection}}{Area_{union}} = \frac{A \cap B}{A \cup B}$$

## 四、科学家故事

### 4.1 Ross Girshick — R-CNN系列的缔造者

[Ross Girshick](https://people.eecs.berkeley.edu/~rbg/)（常被称为RBG）是目标检测领域最具影响力的研究者之一。他在UC Berkeley攻读博士期间，于2014年提出了[R-CNN](https://arxiv.org/abs/1311.2524)，首次将卷积神经网络成功应用于目标检测任务。

**关键贡献时间线**：
- **2014年**：R-CNN — 将CNN特征与选择性搜索结合，在PASCAL VOC上提升30% mAP
- **2015年**：Fast R-CNN — 引入ROI Pooling，实现端到端训练
- **2015年**：Faster R-CNN — 与何恺明合作提出RPN，首次实现实时检测
- **2018年**：Mask R-CNN — 扩展到实例分割，获得ICCV最佳论文奖

Girshick的研究风格以**简洁优雅**著称，他善于将复杂问题分解为可解决的子问题。他曾说："好的研究不是添加更多复杂性，而是找到正确的简化方式。"

### 4.2 Joseph Redmon — YOLO之父

[Joseph Redmon](https://pjreddie.com/) 是YOLO（You Only Look Once）系列的创始人，他于2015年提出的YOLO v1彻底改变了目标检测的范式——从"两阶段检测"（先提议后分类）转向"单阶段检测"（直接预测）。

**YOLO的创新之处**：

Redmon在2020年因伦理担忧宣布退出计算机视觉研究，他在声明中说："我不再参与任何与军事应用或侵犯隐私相关的计算机视觉研究。"这一决定引发了学术界对AI伦理的广泛讨论。

### 6.2 实验二：OpenCV选择性搜索演示

**实验目的**：理解选择性搜索如何生成高质量区域提议

```python
import cv2

# 加载图像
image = cv2.imread('image.jpg')

# 创建选择性搜索对象
ss = cv2.ximgproc.segmentation.createSelectiveSearchSegmentation()
ss.setBaseImage(image)

# 使用快速模式或质量模式
ss.switchToSelectiveSearchFast()  # 约2000个提议
# ss.switchToSelectiveSearchQuality()  # 约10000个提议

# 运行选择性搜索
rects = ss.process()
print(f"生成 {len(rects)} 个区域提议")

# 可视化前100个提议
for i, rect in enumerate(rects[:100]):
    x, y, w, h = rect
    cv2.rectangle(image, (x, y), (x+w, y+h), (0, 255, 0), 1)

cv2.imshow("Selective Search", image)
cv2.waitKey(0)
```

## 七、趣味小故事

### 7.1 ImageNet竞赛的"意外"突破

2012年，[Alex Krizhevsky](https://en.wikipedia.org/wiki/Alex_Krizhevsky)在[ImageNet大规模视觉识别挑战赛(ILSVRC)](http://www.image-net.org/challenges/LSVRC/)中使用AlexNet，以远超第二名的成绩夺冠。有趣的是，他的成功部分归功于一个"硬件优势"——他使用了两个[GTX 580 GPU](https://en.wikipedia.org/wiki/GeForce_500_series)。当时大多数研究者还在使用CPU训练模型，而Krizhevsky的导师[Hinton](https://en.wikipedia.org/wiki/Geoffrey_Hinton)坚持让他尝试GPU计算。这个决定让训练时间从数周缩短到数天，最终催生了深度学习革命。

### 7.2 YOLO的"一夜成名"

Joseph Redmon在2015年CVPR会议前夜还在调试YOLO的代码。当他终于让模型跑通时，发现YOLO不仅在速度上远超当时的方法，精度也相当不错。他兴奋地在Twitter上发布了检测结果视频，一夜之间获得了数千次转发。YOLO的名字也充满个性——Redmon说："我想要一个听起来很酷的名字，You Only Look Once就像是一句特工台词。"

### 7.3 PASCAL VOC的"20类约定"

[PASCAL VOC](http://host.robots.ox.ac.uk/pascal/VOC/)数据集选择20个类别并非随意决定。组织者Mark Everingham等人经过仔细考虑，选择了日常生活中最常见、最具代表性的物体类别。有趣的是，"猫"和"狗"被分开列为两个类别，因为它们是宠物检测中最常见的目标。而"人"类别则成为所有类别中最难检测的——因为人的姿态变化极大，且经常相互遮挡。

## 八、思考题

### 基础题

1. **滑动窗口检测的计算复杂度**：对于一个1024×768的图像，使用64×64的窗口，步长为16，需要评估多少个窗口？如果增加3个尺度（0.5x, 1x, 2x），窗口总数是多少？

2. **IoU计算**：给定两个边界框 $A=(10, 10, 50, 50)$ 和 $B=(30, 30, 70, 70)$（格式：x, y, width, height），计算它们的IoU。

3. **锚框设计**：在RPN中，如果在特征图的每个位置设置3个尺度和3个长宽比的锚框，特征图大小为50×38，总共会产生多少个锚框？其中大约有多少个会被标记为正样本（假设每张图有10个目标）？

### 进阶题

4. **选择性搜索 vs RPN**：比较选择性搜索和RPN在速度、召回率和可学习性方面的差异。为什么Faster R-CNN能够将检测速度提升10倍以上？

5. **NMS阈值选择**：讨论NMS中IoU阈值的选择对检测结果的影响。阈值过高或过低分别会导致什么问题？如何设计自适应NMS策略？

6. **锚框尺度设计**：RPN中锚框的尺度是基于原图还是特征图设计的？如果输入图像大小变化，锚框需要如何调整？

### 实践题

7. **实现NMS**：用Python实现非极大值抑制算法，并在给定的检测框上测试。

8. **性能对比**：使用OpenCV分别运行Haar级联分类器、HOG+SVM和DNN模块的Faster R-CNN，比较它们在相同图像上的检测速度和精度。
