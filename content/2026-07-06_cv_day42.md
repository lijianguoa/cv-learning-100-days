## 二、历史背景

### 目标检测范式的演变

目标检测领域在深度学习时代经历了三次重大范式转变：

1. **两阶段检测时代（2014-2017）**：以 [R-CNN](https://arxiv.org/abs/1506.01497) 系列为代表，先生成候选区域再分类。精度高但速度慢，流程复杂。
2. **单阶段检测时代（2016-2020）**：以 [YOLO](https://arxiv.org/abs/1506.02640)、[SSD](https://arxiv.org/abs/1512.02325) 为代表，直接在特征图上预测。速度快但需要手动设计锚框（Anchor）和NMS后处理。
3. **端到端Transformer检测时代（2020-至今）**：以 [DETR](https://arxiv.org/abs/2005.12872) 为开端，彻底消除了锚框、NMS等手工组件，将目标检测简化为集合预测问题。

### DETR的诞生

2020年5月，**Facebook AI Research (FAIR)** 发布了 [DETR (DEtection TRansformer)](https://arxiv.org/abs/2005.12872)，这篇由 Nicolas Carion 等人撰写的论文被誉为目标检测领域的里程碑之作。DETR首次将 Transformer 的 encoder-decoder 架构成功应用于目标检测，实现了真正的端到端训练。

### Deformable DETR的改进

2021年，**中科院自动化所（CASIA）** 与 FAIR 合作提出了 [Deformable DETR](https://arxiv.org/abs/2010.04159)，由 Xizhou Zhu 等人完成，针对 DETR 训练慢、小目标性能差等核心问题进行了突破性改进，使 Transformer 检测器的实用性大幅提升。

## 三、核心概念

### 3.1 DETR整体架构

DETR 的核心思想是将目标检测视为一个 **集合预测问题（Set Prediction）**，整体流程如下：

```
输入图像 → Backbone (ResNet-50) → Transformer Encoder → Transformer Decoder → 预测集合（N个边界框+类别）
```

**三大核心组件**：

1. **CNN Backbone**：使用预训练的 ResNet-50 提取图像特征，生成特征图（如 1000×2048 的特征）
2. **Transformer Encoder**：将特征图展平并加入位置编码，通过多头自注意力捕获全局依赖
3. **Transformer Decoder**：使用 N 个可学习的 Object Queries，通过交叉注意力从编码器输出中提取目标信息

### 3.2 二分图匹配（Bipartite Matching）

DETR 的核心创新之一是使用 **匈牙利算法（Hungarian Algorithm）** 进行预测框与真实框的一对一匹配。

参考：[Hungarian Algorithm - Wikipedia](https://en.wikipedia.org/wiki/Hungarian_algorithm)

### 3.3 Object Queries（对象查询）

Object Queries 是 DETR 中最关键的设计之一：
- 共 N 个可学习嵌入（默认 N=100），每个 Query 负责预测一个目标
- 训练时通过匈牙利匹配自动学习关注哪些位置的目标
- N 需要大于图像中最多可能出现的目标数量（多余的预测为"无目标"类）

### 3.4 Set Prediction Loss

DETR 的总损失为分类损失 + L1损失 + GIoU损失。

### 3.5 Deformable DETR的关键改进

| 问题 | DETR | Deformable DETR |
|:---|:---|:---|
| **收敛速度** | 500 epochs | 50 epochs（快10倍） |
| **注意力计算** | 全局注意力（O(n²)） | 多尺度可变形注意力（O(nk)，k为采样点数） |
| **小目标性能** | 较差 | 显著提升 |
| **特征尺度** | 单尺度 | 多尺度特征金字塔 |

### 3.6 多尺度可变形注意力（Multi-Scale Deformable Attention）

每个注意力头只关注特征图上的 **K 个采样点**（默认 K=4），而非所有位置：
- 采样位置通过可学习的偏移量预测
- 不同尺度层之间共享 Query
- 计算复杂度从 O(n²) 降至 O(nk)

参考：[Deformable Convolutional Networks](https://arxiv.org/abs/1703.06211)

## 六、经典实验

### DETR vs Faster R-CNN

| 模型 | Backbone | AP | FPS |
|:---|:---|:---|:---|
| Faster R-CNN | ResNet-50 | 42.0 | 26 |
| DETR | ResNet-50 | 42.0 | 28 |
| DETR | ResNet-50-DC5 | 43.4 | 22 |
| DETR | ResNet-101-DC5 | 44.9 | 18 |

### 收敛速度对比

| 模型 | 达到40 AP所需Epochs | GPU时间 |
|:---|:---|:---|
| DETR | ~300 | 6天 |
| Deformable DETR | ~25 | 16小时 |
| Faster R-CNN | ~12 | 10小时 |

参考：[COCO Detection Evaluation](https://cocodataset.org/#detection-eval)

## 七、小故事

### "不需要NMS"的震撼时刻

DETR 用一个优雅的匈牙利匹配替代了整个 NMS 流程。

### "训练500个epoch"的槽点

这个痛点直接催生了 Deformable DETR 的诞生，将训练速度提升了 10 倍。
