## 2. 历史背景 | Historical Background

### 2.1 几何变换的数学起源

几何变换的数学基础可以追溯到19世纪初。1827年，德国数学家**奥古斯特·费迪南德·莫比乌斯**（August Ferdinand Möbius）首次提出了**齐次坐标系**（Homogeneous Coordinates）的概念，为投影几何和仿射变换奠定了数学基础。莫比乌斯还创造了著名的莫比乌斯环（Möbius Strip）。

### 2.2 计算机图形学的发展

- **1963年**：Larry Roberts发表了计算机视觉领域的第一篇专业论文，研究简单几何体的边缘提取和三维重建
- **1970年代**：计算机图形学开始发展，几何变换成为核心算法
- **1999年**：Intel启动OpenCV项目，将几何变换等算法开源
- **2000年至今**：GPU加速使实时几何变换成为可能

### 2.3 OpenCV中的几何变换

OpenCV从1.0版本开始就提供了完整的几何变换支持。`cv2.warpAffine`和`cv2.warpPerspective`是两个核心函数，分别用于仿射变换和透视变换。

## 3. 核心概念 | Core Concepts

### 3.1 图像绘制基础

OpenCV提供了丰富的绘图函数，所有绘图函数都直接在图像上操作：

| 函数 | 功能 | 关键参数 |
|------|------|----------|
| `cv2.line()` | 绘制线段 | 起点、终点、颜色、线宽 |
| `cv2.rectangle()` | 绘制矩形 | 左上角、右下角、颜色、线宽 |
| `cv2.circle()` | 绘制圆形 | 圆心、半径、颜色、线宽 |
| `cv2.ellipse()` | 绘制椭圆 | 中心、轴长、角度、颜色 |
| `cv2.polylines()` | 绘制多边形 | 顶点数组、是否闭合、颜色 |
| `cv2.putText()` | 绘制文字 | 文本、位置、字体、大小、颜色 |

**颜色格式注意**：OpenCV使用BGR格式而非RGB！
- 红色 = `(0, 0, 255)`
- 绿色 = `(0, 255, 0)`
- 蓝色 = `(255, 0, 0)`

### 3.2 图像缩放 (Scaling)

缩放是改变图像尺寸的基本操作：

```python
import cv2

# 方法1：指定目标尺寸
resized = cv2.resize(img, (width, height), interpolation=cv2.INTER_LINEAR)

# 方法2：指定缩放因子
resized = cv2.resize(img, None, fx=0.5, fy=0.5, interpolation=cv2.INTER_AREA)
```

**插值方法选择**：
- `cv2.INTER_NEAREST`：最近邻插值（最快，质量低）
- `cv2.INTER_LINEAR`：双线性插值（默认，平衡）
- `cv2.INTER_CUBIC`：双三次插值（质量高，较慢）
- `cv2.INTER_AREA`：区域插值（缩小图像时推荐）

### 3.3 图像平移 (Translation)

平移是将图像在x和y方向上移动：

```python
import cv2
import numpy as np

# 定义平移矩阵 [1, 0, tx], [0, 1, ty]
M = np.float32([[1, 0, 100], [0, 1, 50]])
translated = cv2.warpAffine(img, M, (width, height))
```

### 3.4 图像旋转 (Rotation)

旋转需要指定旋转中心和角度：

```python
# 获取旋转矩阵
height, width = img.shape[:2]
center = (width // 2, height // 2)
M = cv2.getRotationMatrix2D(center, angle=45, scale=1.0)

# 应用旋转
rotated = cv2.warpAffine(img, M, (width, height))
```

### 3.5 仿射变换 (Affine Transformation)

仿射变换是线性变换与平移的组合，保持**平行性**（平行线变换后仍平行）：

**数学表达**：
```
[x']   [a11 a12 a13] [x]
[y'] = [a21 a22 a23] [y]
[1 ]   [ 0   0   1 ] [1]
```

**仿射变换包含**：
- 平移（Translation）
- 缩放（Scaling）
- 旋转（Rotation）
- 剪切（Shearing）

```python
# 定义三个对应点（原图 -> 目标图）
pts1 = np.float32([[50, 50], [200, 50], [50, 200]])
pts2 = np.float32([[10, 100], [200, 50], [100, 250]])

# 计算仿射矩阵
M = cv2.getAffineTransform(pts1, pts2)
affine = cv2.warpAffine(img, M, (width, height))
```

### 3.6 透视变换 (Perspective Transformation)

透视变换（也称投影变换）可以模拟3D到2D的投影效果，**不保持平行性**：

```python
# 定义四个对应点
pts1 = np.float32([[56, 65], [368, 52], [28, 387], [389, 390]])
pts2 = np.float32([[0, 0], [300, 0], [0, 300], [300, 300]])

# 计算透视矩阵
M = cv2.getPerspectiveTransform(pts1, pts2)
perspective = cv2.warpPerspective(img, M, (300, 300))
```

**应用场景**：
- 文档扫描（将倾斜文档矫正为矩形）
- 增强现实（将虚拟物体投影到真实场景）
- 全景图像拼接

### 实验2：文档扫描矫正

```python
import cv2
import numpy as np

def scan_document(image_path):
    # 读取图像
    img = cv2.imread(image_path)
    
    # 转换为灰度图
    gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)
    
    # 边缘检测
    edges = cv2.Canny(gray, 50, 150)
    
    # 查找轮廓
    contours, _ = cv2.findContours(edges, cv2.RETR_EXTERNAL, 
                                    cv2.CHAIN_APPROX_SIMPLE)
    
    # 找到最大的四边形（假设是文档）
    for cnt in contours:
        peri = cv2.arcLength(cnt, True)
        approx = cv2.approxPolyDP(cnt, 0.02 * peri, True)
        
        if len(approx) == 4:
            # 获取四个角点
            pts1 = np.float32([approx[i][0] for i in range(4)])
            
            # 计算目标矩形尺寸
            width = max(np.linalg.norm(pts1[0] - pts1[1]),
                       np.linalg.norm(pts1[2] - pts1[3]))
            height = max(np.linalg.norm(pts1[0] - pts1[3]),
                        np.linalg.norm(pts1[1] - pts1[2]))
            
            # 定义目标点
            pts2 = np.float32([[0, 0], [width, 0], 
                              [width, height], [0, height]])
            
            # 透视变换
            M = cv2.getPerspectiveTransform(pts1, pts2)
            scanned = cv2.warpPerspective(img, M, (int(width), int(height)))
            
            return scanned
    
    return None
```

## 7. 小故事 | Interesting Stories

### 7.1 为什么OpenCV用BGR而不是RGB？

这是一个历史遗留问题。在OpenCV诞生时（1999年），当时最流行的相机制造商和软件库使用BGR格式。Intel为了兼容性选择了BGR，尽管现在RGB更常见，但为了向后兼容，OpenCV一直保持着这个"奇怪"的约定。

### 7.2 透视变换与文艺复兴艺术

透视变换的原理与文艺复兴时期的画家们发现的一样——物体在远处看起来更小。15世纪的意大利建筑师布鲁内莱斯基（Filippo Brunelleschi）首次系统性地研究了透视画法。今天，我们用数学矩阵来实现500年前艺术家们的发现！

### 7.3 齐次坐标的妙用

齐次坐标`(x, y, 1)`看起来只是多加了一个1，但它让平移也能用矩阵乘法表示！没有齐次坐标，平移只能用加法，而仿射变换就无法统一用矩阵乘法表达了。这就是为什么莫比乌斯1827年的发明至今仍在GPU中每天运行数十亿次。

## 9. 术语表 | Glossary

| 术语 | 英文 | 定义 |
|------|------|------|
| 仿射变换 | Affine Transformation | 保持平行性的线性变换与平移的组合 |
| 透视变换 | Perspective Transformation | 模拟3D投影效果的变换，不保持平行性 |
| 齐次坐标 | Homogeneous Coordinates | 增加一个维度的坐标表示法，使平移可用矩阵乘法表示 |
| 插值 | Interpolation | 根据已知像素值估计未知像素值的方法 |
| 单应性矩阵 | Homography | 描述两个平面之间投影变换的3×3矩阵 |
| 剪切变换 | Shearing | 使图像沿某一方向倾斜的变换 |
| 边缘扩展 | Padding | 在图像边缘添加像素以扩展尺寸的操作 |
